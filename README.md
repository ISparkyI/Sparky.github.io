# Портфоліо проєктів

Багатосторінковий сайт-портфоліо без збірки — просто HTML/CSS/JS. Дві сторінки:

- **index.html** — головна: hero + проєкти, розділені на категорії з фільтром
- **contact.html** — форма для звʼязку щодо співпраці (лист приходить вам на пошту)

Темна тема, червоний акцент, анімований фон із частинками, що реагує на курсор. Інтерфейс — англійською за замовчуванням, з перемикачем EN/UK у правому верхньому куті.

## Файли

| Файл | Що це |
|---|---|
| `index.html` | Головна сторінка |
| `contact.html` | Сторінка з формою зв'язку |
| `style.css` | Усі стилі (спільні для обох сторінок) |
| `script.js` | Переклад інтерфейсу, анімація фону, рендер/фільтр проєктів, відправка форми |
| `projects-data.js` | **Тут редагуєте список своїх проєктів** |

## Як додати свої проєкти

Відкрийте `projects-data.js` — там масив `PROJECTS`. Скопіюйте один об'єкт і заповніть свої дані:

```js
{
  title: "Sprunki: Knockout",
  category: "games",          // МАЄ бути одне з: games, apps, web, other
  description: "Short description of the game.",
  tags: ["Unity", "C#"],
  image: "images/sprunki.jpg", // необов'язково — див. розділ про картинки нижче
  demo: "https://play.google.com/store/apps/details?id=...",
  repo: "",                    // порожньо = кнопка "Code" не показується
},
```

**Важливо про поле `category`** — на відміну від попередньої версії, тут не можна писати будь-яке слово (типу "Ігри" чи "Games впереміш"). Використовуйте рівно один із чотирьох ключів: `"games"`, `"apps"`, `"web"`, `"other"`. Саме з них скрипт автоматично будує:
- кнопки фільтрів на головній сторінці (з правильним перекладом EN/UK),
- напис на головній кнопці картки: `games` → "Play" / "Грати", `apps` → "Try it" / "Спробувати", `web` → "Visit" / "Відкрити", `other` → "View" / "Переглянути".

**Поле `demo`** — посилання, куди веде головна кнопка (Play Store, сайт, itch.io тощо). Якщо ще немає посилання — залиште `""`, кнопка все одно покажеться, просто нікуди не вестиме, поки не заповните.

**Поле `repo`** — посилання на код. Якщо залишити `""` — кнопка "Code"/"Код" просто не з'явиться на картці (зручно для ігор/застосунків без відкритого коду).

⚠️ Назви полів (`title`, `category`, `description`, `tags`, `image`, `demo`, `repo`) — фіксовані, писати їх треба **рівно так**, як у прикладі (маленькими літерами). Якщо написати, наприклад, `Play` замість `demo` — скрипт це поле просто не побачить і кнопка не працюватиме.

### Як додати картинку до картки проєкту

1. Створіть у репозиторії (поряд з `index.html`) папку `images`.
2. Покладіть туди файл, наприклад `sprunki.jpg` (зробіть скриншот гри/сайту або збережіть банер).
3. У `projects-data.js` в полі `image` вкажіть відносний шлях: `"images/sprunki.jpg"`.

Якщо `image` залишити порожнім (`""`), замість фото покажеться нейтральна плашка з назвою категорії — це нормально, картка не зламається.

Можна також вказати пряме посилання на картинку в інтернеті (`"https://..."`), але надійніше зберігати картинки у себе в репозиторії — зовнішні посилання іноді перестають працювати.

## Перемикач мови (EN / UK)

У шапці сайту зверху праворуч є дві кнопки — EN і UK. Вони перемикають **інтерфейс** сайту (написи кнопок, заголовки, форму) і запам'ятовують вибір у браузері відвідувача.

Важливо: **назви й описи ваших проєктів мова не перекладає** — це ваш власний текст у `projects-data.js`, він показується як є, незалежно від обраної мови інтерфейсу. Якщо хочете, щоб і описи проєктів були двомовні — пишіть мені, можна додати таку можливість окремо.

## Як підключити відправку форми на пошту (безкоштовно)

Сайт на GitHub Pages — статичний, без сервера, тому форма використовує безкоштовний сервіс [Formspree](https://formspree.io). У вас це вже підключено (ID `xzdlygko` вже стоїть у `contact.html`) — нічого додатково робити не треба, якщо тільки не захочете створити іншу форму.

Якщо колись знадобиться підключити наново:
1. Зареєструйтесь на formspree.io (можна через GitHub-акаунт).
2. Створіть нову форму (**New Form**) і вкажіть пошту, на яку мають приходити повідомлення.
3. Formspree дасть посилання виду `https://formspree.io/f/xxxxxxxx` — скопіюйте його.
4. У файлі `contact.html` замініть значення `action="https://formspree.io/f/..."` на нове.
5. Formspree надішле лист-підтвердження на вашу пошту — підтвердіть її, інакше форма не запрацює.
6. Безкоштовний план дозволяє 50 повідомлень на місяць.

## Кольори та стиль

Усі кольори — у самому верху `style.css`, у блоці `:root`. Головний акцент:
```css
--accent: #e6394a;
```
Змініть це значення, якщо захочете інший відтінок червоного.

Анімований фон (частинки) налаштовується на початку `script.js` у секції "ANIMATED BACKGROUND": можна змінити кількість частинок (`PARTICLE_COUNT`), відстань зв'язків (`LINK_DIST`) і радіус реакції на курсор (`MOUSE_DIST`).

## Як опублікувати на GitHub Pages

1. Створіть репозиторій на GitHub і зробіть його **публічним** (Settings → General → Danger Zone → Change visibility), інакше розділ Pages буде недоступний на безкоштовному акаунті.
2. Завантажте всі файли (і папку `images`, якщо створили) з цієї папки в репозиторій.
3. Settings → Pages → Source: **Deploy from a branch**, гілка `main`, папка `/root` → Save.
4. Через 1–2 хвилини сайт з'явиться за посиланням, яке GitHub покаже на цій сторінці.

## Локальний перегляд

Просто відкрийте `index.html` у браузері.
